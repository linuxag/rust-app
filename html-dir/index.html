Terraform Basics, Providers, Variables,

Rahul Maurya Terraform December 12, 2021December 10, 2021 2 Minutes
  • Terraform works on plugin based architecture.
  • whenever we run terraform init, it downloads and installs the plugins based on the provider that we are using. it shows the plugin version as well.
  • plugins are downloaded as a hidden file(.terraform/plugin) in the working directry.
  • Providers are like aws, azure, gcp, local_file, etc…Like this there are 100’s of providers
  • terraform providers are available on registry.terraform.io

Types of Providers?
Majorly providers are divided into 3 types.

  1. Official Provider: These are maintained by Hashicorp. official providers are aws, GCP, Azure, Local_file etc
  2. Verified Provider: These are maintained by third party vendors. bigip is maintained by F5 Networks, heroku by heroku, digitalocean by digitalocean etc.
  3. Community Provider: These are maintained by community. activedirectory, ucloud, netapp-gcp etc

How to use Multiple providers and resources in a single configuration file?

resource "local_file" "my-pet" {
    filename = "/root/pet-name"
    content = "My pet is called finnegan!!"
}

resource "random_pet" "other-pet" {
    length = "1"
    prefix = "Mr"
    separator = "."
}

How Take Variable as Input?

Filename: variable.tf

variable "filename" {
  default     = "/root/pets.txt"
}
variable "content" {
  default     = "We love Pets"
}
variable "prefix" {
  default     = "Mrs"
}

Here filename, content, and prefix are the variables and their default values are also given. Default values are optional.

Now we can use the above variable in any configuration file as below.
filename: main.tf

resource "local_file" "my-pet" {
    filename = var.filename
    content = var.content
}

resource "random_pet" "other-pet" {
    length = "1"
    prefix = var.prefix
}

Now run terraform init, plan and apply and see the result.

var.filename will take the default value that we specified.

another example of variable is as below.

variable "ami" {
  default     = "ami-0e4wf54efj694jd43"
}

variable "instance_type" {
  default     = "t2.micro"
}

resource "aws_instance" "webserver" {
    ami = var.ami
    instance_type = var.instance_type
}

By default variables take 3 inputs as follows

variable name {
  type        = string
  default     = ""
  description = "description"
}

there are various types os variables that we can take as an input like string, number, bool, any, list, map, object etc.

List Variables

variable "prefix" {
  type        = list
  default     = ["Mr", "Mrs", "Sir"]
}

Now how to use the List variable?

resource "random_pet" "my-pet" {
    prefix = var.prefix[0]
}

Prefix[0] means it will take first default value as a variable from the list.

We can also specify list as number or string like bwlow.

variable "prefix" {
  type        = list(string)
  default     = ["Mr", "Mrs", "Sir"]
}

variable "prefix" {
  type        = list(number)
  default     = ["1", "2", "3"]
}

What is Output Variables?

output "pet-name" {
  value       = "cat"
}

now again run init, plan and apply.

We can view the terraform output variable by below command

terraform output
